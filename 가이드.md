# han-auto 보고서 생성 가이드

이 문서는 브라더 공공기관 보고서 양식을 사용해 `한라대_2026전국동시지방선거.hwp` 원문을 표 포함 HWPX 보고서로 만든 과정을 기준으로 정리한 작업 가이드입니다.

## 1. 목적

`han-auto`는 HWP/HWPX 양식을 유지한 채 공공기관 보고서 초안을 자동 생성하는 도구입니다.

주요 기능은 다음과 같습니다.

- HWP/HWPX 보고서 양식에 제목, 목차, 본문을 자동 삽입
- PDF/TXT/Markdown/HWP/HWPX 참고자료에서 원문 텍스트 추출
- 주제와 원문을 바탕으로 4개 장 보고서 초안 생성
- 예산, 일정, 역할분담 같은 내용을 실제 HWPX 표로 삽입
- HWPX 내부의 로고 이미지를 새 이미지로 교체

## 3. 생성 흐름

보고서 생성 흐름은 다음 순서입니다.

1. HWP 원문을 HWPX로 임시 변환한다.
2. HWPX 내부 `Contents/section*.xml`에서 텍스트를 추출한다.
3. 원문 텍스트를 `generate_report_draft()`에 넘겨 보고서 초안을 만든다.
4. 초안의 `sections`는 본문 4개 장으로 렌더링한다.
5. 초안의 `tables`는 실제 HWPX 표 XML로 렌더링한다.
6. 필요하면 `--logo` 또는 `logo_path`로 로고 이미지를 교체한다.
7. 결과 HWPX를 `artifacts` 폴더에 저장한다.

관련 코드:

```text
src\han_auto\source.py       원문 텍스트 추출
src\han_auto\draft.py        보고서 초안/표 데이터 생성
src\han_auto\hwpx_report.py  HWPX 양식 렌더링, 표 삽입, 이미지 교체
src\han_auto\cli.py          CLI 명령어
```

## 5. Python으로 직접 생성하는 방법

개발 환경에서 CLI 실행이 막히면 아래 방식으로 직접 실행할 수 있습니다.

```powershell
$env:HAN_AUTO_SOURCE_PATH = "C:\Users\jjmjh\Documents\카카오톡 받은 파일\한라대_2026전국동시지방선거.hwp"
$env:HAN_AUTO_TEMPLATE_PATH = "C:\Users\jjmjh\Downloads\브라더 공공기관 보고서 양식.hwpx"
$env:HAN_AUTO_OUTPUT_PATH = "C:\Users\jjmjh\Documents\han-auto\artifacts\한라대_2026전국동시지방선거_AI선거보도_보고서.hwpx"

@'
import os
from pathlib import Path
from han_auto.source import extract_source_text
from han_auto.draft import generate_report_draft
from han_auto.hwpx_report import render_public_report_hwpx

source = Path(os.environ["HAN_AUTO_SOURCE_PATH"])
template = Path(os.environ["HAN_AUTO_TEMPLATE_PATH"])
output = Path(os.environ["HAN_AUTO_OUTPUT_PATH"])

notes = extract_source_text(source)
draft = generate_report_draft(
    topic="2026 전국동시지방선거 AI 선거보도 협업",
    company="한라대학교 AI인공지능학과",
    audience="KBS제주방송총국 및 선거보도 실무진",
    notes=notes,
    provider="offline",
)

render_public_report_hwpx(
    template_path=template,
    output_path=output,
    draft=draft,
    resave_with_hwp=False,
)

print(output)
'@ | python -
```

## 6. 표 포함 초안 작성 기준

초안 데이터는 `ReportDraft` 구조를 사용합니다.

중요 필드:

```text
title       문서 제목
date        작성일
company     작성 주체
sections    본문 4개 장
tables      실제 HWPX 표로 렌더링할 표 목록
attachments 붙임 항목
references  참고 항목
```

표 데이터는 다음 구조입니다.

```json
{
  "title": "예산 산출 내역",
  "columns": ["항목", "금액", "비고"],
  "rows": [
    ["AI 현안 보도 제작 지원", "3,000,000원", "32개 선거구 현안·공약 초안 작성"],
    ["인터랙티브 페이지 개설", "2,000,000원", "선거정보 페이지 구축"],
    ["관리운영", "2,000,000원", "2026.5.6.~6.3. 운영"]
  ],
  "note": "부가세와 세부 정산 기준은 계약 협의 시 별도 확정한다."
}
```

이번 보고서에는 다음 3개 표를 넣었습니다.

- 예산 산출 내역
- 추진 일정
- 역할분담

표는 텍스트로 흉내낸 것이 아니라 HWPX의 `hp:tbl`, `hp:tr`, `hp:tc` 구조로 들어갑니다.

## 7. 표 서식 가이드

현재 표 서식은 `src\han_auto\hwpx_report.py`에서 관리합니다.

주요 상수:

```python
TABLE_WIDTH = 47688
TABLE_ROW_HEIGHT = 2200
TABLE_CELL_MARGIN = 141
TABLE_TITLE_CHAR_PR_ID = "36"
TABLE_HEADER_CHAR_PR_ID = "37"
TABLE_BODY_CHAR_PR_ID = "38"
```

적용된 스타일:

```text
표 제목: 13pt, 검정, 굵게
표 헤더: 10pt, 흰색, 굵게
표 본문: 9pt, 검정
행 높이: 2200
```

헤더 셀은 어두운 배경을 사용하므로 글자는 반드시 흰색이어야 합니다.

확인 기준:

```text
항목 borderFill 9 charPrIDRef 37 textColor #FFFFFF
금액 borderFill 9 charPrIDRef 37 textColor #FFFFFF
비고 borderFill 9 charPrIDRef 37 textColor #FFFFFF
```

표 헤더가 어둡게 보이는데 글자가 검게 보이면 `TABLE_HEADER_CHAR_PR_ID`가 HWPX `header.xml`에 제대로 추가되었는지 확인합니다.

## 8. 이미지/로고 변경 가이드

### 8.1 CLI에서 로고 교체

보고서 양식 안의 기존 로고 이미지를 바꾸려면 `--logo` 옵션을 사용합니다.

```powershell
han-auto draft-hwpx `
  "C:\path\to\template.hwpx" `
  --topic "보고서 주제" `
  --company "회사명 또는 기관명" `
  --source "C:\path\to\source.hwp" `
  --logo "C:\path\to\logo.png" `
  --output "C:\path\to\output.hwpx"
```

지원 이미지:

```text
PNG, JPG 등 Pillow가 열 수 있는 이미지
```

내부 동작:

1. `logo_path` 이미지를 Pillow로 연다.
2. 투명 영역이나 빈 여백을 가능한 만큼 잘라낸다.
3. HWPX 내부 첫 번째 `hp:pic` 그림 정보를 새 이미지 크기에 맞춘다.
4. `BinData/image1.png`를 새 PNG 이미지로 교체한다.

관련 코드:

```text
src\han_auto\hwpx_report.py
_replace_logo()
_prepare_logo_png()
```

### 8.2 로고 이미지 권장 조건

권장 형식:

```text
PNG
투명 배경
가로형 로고
가로 600px 이상
```

주의할 점:

- 너무 작은 이미지는 HWP에서 확대되어 흐려질 수 있습니다.
- 흰 배경 로고는 양식 배경과 구분이 안 될 수 있습니다.
- 로고 주변 여백은 자동으로 어느 정도 잘리지만, 원본 자체를 깔끔하게 준비하는 것이 좋습니다.

### 8.3 현재 구현의 한계

현재 로고 교체는 HWPX 안의 첫 번째 그림을 대상으로 합니다.

즉, 템플릿에 여러 이미지가 있을 때 특정 이미지를 이름으로 골라 교체하는 기능은 아직 없습니다.

여러 이미지를 정확히 바꾸려면 다음 기능을 추가해야 합니다.

- HWPX 내부 `hp:pic` 목록 조회
- 그림 파일명 또는 위치 기준 선택
- `BinData/imageN.*`와 manifest 관계 정보 갱신
- 여러 이미지 동시 교체 옵션

현재 양식의 대표 로고 교체에는 `--logo` 옵션으로 충분합니다.

## 9. HWP/HWPX 원문 처리 가이드

`--source`는 다음 형식을 지원합니다.

```text
.pdf
.txt
.md
.hwp
.hwpx
```

`.hwp` 원문은 바로 텍스트를 읽지 않고 다음 과정을 거칩니다.

1. `hwp2hwpx` Java 라이브러리로 임시 `.hwpx`로 변환
2. 변환된 HWPX의 `Contents/section*.xml`에서 텍스트 추출
3. 공백 정리 후 초안 생성에 사용

처음 `.hwp` 변환 시 필요한 항목:

```text
JDK
neolord0/hwp2hwpx source
hwplib jar
hwpxlib jar
```

이 프로젝트는 `.tools` 폴더에 변환 도구 캐시를 저장합니다.

```powershell
$env:HAN_AUTO_OUTPUT_PATH = "C:\Users\jjmjh\Documents\han-auto\artifacts\한라대_2026전국동시지방선거_AI선거보도_보고서_표가독성조정.hwpx"

@'
from pathlib import Path
import os, zipfile, xml.etree.ElementTree as ET

path = Path(os.environ["HAN_AUTO_OUTPUT_PATH"])
with zipfile.ZipFile(path) as zf:
    section = ET.fromstring(zf.read("Contents/section0.xml"))
    header = ET.fromstring(zf.read("Contents/header.xml"))

ns = {"hp": "http://www.hancom.co.kr/hwpml/2011/paragraph"}
HH = "{http://www.hancom.co.kr/hwpml/2011/head}"
char_prs = {node.attrib["id"]: node.attrib for node in header.findall(f".//{HH}charPr")}

tables = section.findall(".//hp:tbl", ns)
first_new_table = tables[-3]
header_cells = first_new_table.findall(".//hp:tc", ns)[:3]

for cell in header_cells:
    text = "".join(node.text or "" for node in cell.findall(".//hp:t", ns))
    run = cell.find(".//hp:run", ns)
    char_id = run.attrib["charPrIDRef"]
    print(text, char_id, char_prs[char_id]["textColor"], char_prs[char_id]["height"])
'@ | python -
```

## 11. 문제 해결

### 출력 파일을 덮어쓸 수 없을 때

증상:

```text
PermissionError: [Errno 13] Permission denied
```

원인:

```text
해당 HWPX 파일이 한컴 HWP에서 열려 있음
```

해결:

- 한컴에서 파일을 닫고 다시 생성합니다.
- 또는 출력 파일명을 바꿉니다.

예:

```text
..._표폰트조정.hwpx
..._표가독성조정.hwpx
```

### HWP 변환이 실패할 때

확인할 것:

- `.tools` 폴더에 JDK와 hwp2hwpx 캐시가 있는지 확인
- 네트워크가 GitHub/Maven/Adoptium 다운로드를 막는지 확인
- `HAN_AUTO_JAVA_HOME`, `HAN_AUTO_HWP2HWPX_SOURCE`, `HAN_AUTO_HWP2HWPX_CLASSPATH` 환경변수로 수동 경로 지정 가능

### 표 헤더 글자가 잘 안 보일 때

확인할 것:

- 헤더 셀 `borderFillIDRef`가 어두운 배경인지 확인
- 헤더 글자 스타일 `charPrIDRef`가 `37`인지 확인
- `header.xml`의 `charPr id="37"`에 `textColor="#FFFFFF"`가 있는지 확인

### 로고가 바뀌지 않을 때

확인할 것:

- `--logo` 경로가 실제 파일인지 확인
- 템플릿 안에 `hp:pic` 그림 요소가 있는지 확인
- 이미지가 너무 큰 경우 PNG로 저장한 뒤 다시 시도

### 12.2 좋은 프롬프트의 원칙

Codex/Claude에게 요청할 때는 다음을 분명히 적습니다.

- 한국어 공공기관 보고서 문체로 작성
- Markdown 설명이나 코드펜스 없이 JSON 객체만 반환
- `sections`는 정확히 4개
- 각 장에는 `groups`를 1~3개 포함
- 예산, 일정, 역할분담, 정량 수치는 본문에 풀어쓰기보다 `tables`에 넣기
- 표의 금액, 기간, 건수에는 단위를 붙인 문자열 사용
- 원문에 없는 수치는 추정하지 말고 `검토 필요` 또는 `협의 후 확정`으로 표시
- HWPX 렌더러가 쓰는 스키마 외 필드는 만들지 않기

### 12.3 Codex/Claude 직접 요청용 프롬프트

아래 프롬프트는 Codex나 Claude 대화창에 직접 넣을 수 있는 형태입니다.

```text
너는 한국 공공기관 보고서와 제안서 작성 전문가다.
아래 원문을 바탕으로 HWPX 공공기관 보고서 양식에 들어갈 초안 JSON만 작성하라.

작성 조건:
- 반드시 한국어로 작성한다.
- 과장된 마케팅 문구보다 공공기관 보고서 문체를 사용한다.
- 반드시 JSON 객체만 반환한다.
- Markdown 코드펜스, 설명 문장, 주석을 쓰지 않는다.
- sections는 정확히 4개로 구성한다.
- 각 section.groups는 1~3개로 구성한다.
- 각 group.points는 1~3개 문장으로 구성한다.
- tables는 1~3개로 구성한다.
- 예산, 일정, 역할분담처럼 수치나 항목 비교가 필요한 내용은 tables에 넣는다.
- 표 안의 금액, 건수, 기간, 비율은 단위를 붙인 문자열로 작성한다.
- 원문에 없는 숫자는 추정하지 않는다.
- 제목, 목차, 본문에 바로 넣을 수 있도록 문장을 짧고 명료하게 작성한다.
```

### 12.5 피해야 할 프롬프트

아래 같은 요청은 결과가 불안정합니다.

```text
이 자료로 보고서 예쁘게 써줘.
표도 적당히 넣어줘.
```

문제점:

- JSON 구조가 보장되지 않습니다.
- 표가 실제 `tables` 구조가 아니라 Markdown 표나 문장으로 나올 수 있습니다.
- 금액과 일정이 본문에 섞여 HWPX 표로 렌더링되지 않을 수 있습니다.
- 모델이 원문에 없는 숫자를 추정할 가능성이 커집니다.

### 12.6 좋은 추가 지시 예시

정책 보고서 톤을 더 강하게 하고 싶을 때:

```text
문체는 공공기관 내부 보고서 톤으로 유지하고, 홍보성 표현은 줄여라.
```

표를 더 촘촘하게 만들고 싶을 때:

```text
tables에는 예산 산출 내역, 추진 일정, 역할분담 3개 표를 반드시 포함하라.
각 표는 3개 열로 구성하고, 행은 3~5개 범위로 작성하라.
```

수치 왜곡을 막고 싶을 때:

```text
원문에 없는 금액, 기간, 선거구 수는 추가하지 마라.
불명확한 항목은 "협의 후 확정"으로 적어라.
```

HWPX에 바로 넣기 좋게 짧게 만들고 싶을 때:

```text
각 points 문장은 45자 안팎으로 짧게 작성하라.
표 셀 문구는 줄바꿈 없이 20자 안팎으로 압축하라.
```

## 13. 추천 작업 순서

1. 원문 파일을 준비한다.
2. 브라더 공공기관 보고서 HWPX 양식을 준비한다.
3. 필요하면 로고 PNG를 준비한다.
4. `draft-hwpx`로 초안을 생성한다.
5. 한컴 HWP에서 결과물을 열어 제목, 목차, 표, 로고를 확인한다.
6. 표 글자 크기와 헤더 가독성을 확인한다.
7. 필요하면 출력 파일명을 바꿔 재생성한다.
